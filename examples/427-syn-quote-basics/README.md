# 427: syn + quote for Proc Macro AST

**Difficulty:** 4  **Level:** Expert

The two crates that make proc macros practical: `syn` parses Rust token streams into a usable AST, `quote` generates Rust code from templates.

## The Problem This Solves

A proc macro receives a raw `TokenStream` — a flat sequence of tokens. You can't usefully inspect or generate code working at that level. You need to know: is this a struct or an enum? What are the field names and types? Does this field have a `#[serde(rename)]` attribute?

`syn` parses a `TokenStream` into a full Rust AST with named structs: `DeriveInput`, `ItemFn`, `Expr`, `Type`, and hundreds more. You navigate the AST with Rust pattern matching — natural, safe, with compile-time exhaustiveness.

`quote!` is the inverse: write Rust code as a template with `#variable` interpolation and `#(repetition)*` patterns, and get a `TokenStream` back. It's the cleanliest way to generate code — you write what the output should look like, not how to construct it token by token.

Together they're the foundation of essentially every non-trivial proc macro in the ecosystem.

## The Intuition

`syn` turns tokens → AST (read code), `quote!` turns template → tokens (write code) — together they're the read/write API for proc macros.

## How It Works in Rust

```rust
use proc_macro::TokenStream;
use quote::quote;
use syn::{parse_macro_input, DeriveInput, Data, Fields};

#[proc_macro_derive(Getters)]
pub fn derive_getters(input: TokenStream) -> TokenStream {
    // syn: parse raw tokens into structured AST
    let input = parse_macro_input!(input as DeriveInput);

    let struct_name = &input.ident;           // Ident: the type name
    let generics = &input.generics;           // Generic parameters

    // Navigate the AST
    let fields = match &input.data {
        Data::Struct(s) => match &s.fields {
            Fields::Named(f) => &f.named,
            _ => panic!("Only named fields supported"),
        },
        _ => panic!("Only structs supported"),
    };

    // quote: generate code with template interpolation
    let getters = fields.iter().map(|f| {
        let name = f.ident.as_ref().unwrap();  // field name as Ident
        let ty = &f.ty;                         // field type
        quote! {
            pub fn #name(&self) -> &#ty {       // #name interpolated
                &self.#name
            }
        }
    });

    // #(#getters)* — expand the iterator of token streams
    quote! {
        impl #struct_name {
            #(#getters)*
        }
    }.into()
}

// Usage:
#[derive(Getters)]
struct User { name: String, age: u32 }

let u = User { name: "Alice".into(), age: 30 };
println!("{}", u.name());  // getter generated by macro
```

**Key `syn` types:**
- `DeriveInput` — top-level parse target for derive macros
- `ItemFn` — for attribute macros on functions
- `Ident` — a name/identifier
- `Type` — a type expression
- `Expr` — an expression

**Key `quote!` patterns:**
- `#name` — interpolate a variable
- `#(#items)*` — repeat over an iterator
- `#(#items),*` — comma-separated repetition

## What This Unlocks

- **Struct field iteration**: Read all fields, their types, their attributes — generate per-field code.
- **Span-preserving errors**: `syn::Error::new(field.span(), "message")` points errors at the right source location.
- **Full AST access**: Generics, lifetimes, where clauses, visibility — all accessible and usable in generation.

## Key Differences

| Concept | OCaml | Rust |
|---------|-------|------|
| AST parsing | `Ppxlib.Ast` (built-in OCaml AST) | `syn` crate — parses `TokenStream` |
| Code generation | `Ppxlib.Ast_builder` | `quote!` macro — template-based |
| Field access | `type_declaration.ptype_manifest` | `DeriveInput` → `Data::Struct` → `Fields::Named` |
| Identifier manipulation | `Longident.Lident name` | `syn::Ident`, `quote::format_ident!` |
| Error with location | `Location.raise_errorf ~loc` | `syn::Error::new(span, msg)` |
